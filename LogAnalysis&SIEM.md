> Defination:

ðŸ”¹ Log Analysis
- Log analysis is the process of collecting, parsing, and interpreting log data generated by systems, networks, and applications to:
  - Detect security incidents
  - Troubleshoot issues
  - Monitor performance and user activity
- Logs are machine-generated records of events

ðŸ”¹ SIEM (Security Information and Event Management)
- A SIEM is a centralized platform that:
  - Collects logs from across the organization (firewalls, endpoints, servers, cloud)
  - Normalizes and correlates data to find related events
  - Generates alerts and reports for analysts

> Knowledge: What You Need to Understand

ðŸ”¹ Types of Logs
- Application Logs: Messages from specific applications, providing insights into their status, errors, warnings, and other operational details.
- Audit Logs: Events, actions, and changes occurring within a system or application, providing a history of user activities and system behavior.
- Security Logs: Security-related events like logins, permission alterations, firewall activities, and other actions impacting system security.
- Server Logs: System logs, event logs, error logs, and access logs, each offering distinct information about server operations.
- System Logs: Kernel activities, system errors, boot sequences, and hardware status, aiding in diagnosing system issues.
- Network Logs: Communication and activity within a network, capturing information about events, connections, and data transfers.
- Database Logs: Activities within a database system, such as queries performed, actions, and updates.
- Web Server Logs: Requests processed by web servers, including URLs, source IP addresses, request types, response codes, and more.

ðŸ”¹ Analysis Context
- Pattern, Corralation, action
- Timeline, Timestamp, Super Timelines
- Common Patterns, Abnormal User Behavior (Multiple failed login attempts, Unusual login times, Geographic anomalies, Frequent password changes, Unusual user-agent strings)
- Intermitern, recurrent, seldom
- Action/Status: failures, errors, start, stop, halt, refuse
- Severity and Status: Fatal,Critical (or crit), Warning (or warn), Alert, Emergency (or emerg), Info (or informational), Debug, Trace, Success, OK 
- Operational States and Events : Abnormal, Shutdown, Restart, Initialize (or init), Connect (or connecting, connection, connected), Disconnect (or disconnected), Timeout, Crash, Hang, Online / Offline, Up / Down
- Problem Descriptions : Exception, Invalid, Illegal, Denied, Rejected, Unauthorized, Failed to (e.g., "Failed to connect", "Failed to load"), Unable, Cannot (or can not, can't), Missing, Corrupt, Unavailable, Error Value, Out of memory, Null Pointer
- System/Security Related : Unauthorized access, Login fail, Authentication fail, SQL Error, Denial of Service (or DoS), Infection, Vulnerability
- What to looking for : The IP address, The timestamp, The HTTP method (POST, GET, or PUT, for example),The URL, The user-agent,
